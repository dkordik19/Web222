When we talking about responsive design, we are talking about a wide range of modern devices that have screens and access to the internet, because Responsive Design is what good developers should do to satisfy all those screens. I mean, nowadays, if you need to create a good web page you must be sure that this page looks great on all devices. There is no need for the user to see all the elements of a web page. The area that visible for a user called a "viewport". Including a “meta viewport” element to the web page helps the browser to show the content of a web page according to the device requesting the page. This technic is very handy and helps developers while working on responsive design.





Learning outcome

While working on my first web store assignment I found out that there are a lot of unexpected challenges. I discover that with the help of “Media Query” we can create a separate design for devices with certain screen resolution and it helps me a lot in my assignment. When working with images it is a good practice to set the width property with the percentage and let the browser determine the best size of the image according to condition. When we need containers being on its own place and not jumping on the screen when we change a screen resolution the “Grid Layout Module” could help with this; it makes the page looks like a grid, so it becomes much easier to design. Also, we can take advantage of already created and tested frameworks provided by different web sites.





Conclusion

Honestly, there are a lot of tools that we can use while working on responsible design, but in my assignment, I mostly use “flex” and “grid”. The lesson I learned from the assignment: you have to think about a Responsive Design from the first line of code. Working on it afterward really bad Idea.
